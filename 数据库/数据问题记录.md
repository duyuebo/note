# 数据库问题记录

## 一. mybatis与JPA选择

* 选择的标准:
  1. 数据库只做数据的保存,不做业务逻辑的约束判断(不设置外键)
  2. 数据操作只涉及到数据层面,不涉及到逻辑(1对多,多对多这种)
  3. 数据结果集转换要方便
  4. 先阶段
* mybaits相关问题
  1. 需要操作xml(不好调试,代码编写不友好)
  2. 分页组件的支持在mapper层面.而mapper的可重用性基本等于没有.
* jpa相关问题
  1. 需要理解对象4种状态
  2. 轻量级使用,不使用一对多,多对多等
  3. 复杂语句写原生sql
