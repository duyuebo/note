# 设计模式

### 软件设计基本原则

1. 找出软件中变化的部分，并将变化的部分封装起来。
2. 面向接口编程，而不是面向实现。
3. 组合优于继承。
4. 依赖倒置原则（依赖于抽象，不依赖于具体实现）
   * 变量不引用具体的实现类
   * 不从具体的类产生类
   * 不重写基类已经实现的方法
